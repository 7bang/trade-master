## 1. 프로젝트 개요

### 1.1 제품명
**거래의장인** (Trade Master)

### 1.2 제품 비전
유통업체 및 도소매 판매점이 거래처와의 모든 거래 내역을 효율적으로 기록하고 관리할 수 있는 모바일 기반 거래장 관리 솔루션

### 1.3 목표 사용자

#### 1차 타겟
- **소규모 도소매 판매점 사장님** (5-50개 거래처 보유)
- 특징:
  - 수기 장부 또는 간단한 엑셀 사용 중
  - 스마트폰 활용에 익숙
  - 거래처와 카카오톡으로 소통
  - 품목별 거래 관리 필요

#### 2차 타겟
- **중소 유통업체 직원** (50-200개 거래처 관리)
- **개인 도매상, 자영업자**

### 1.4 핵심 가치 제안

| 가치 | 설명 | 달성 목표 |
|------|------|----------|
| 📱 **언제 어디서나** | 모바일로 즉시 거래 기록 | 10초 이내 입력 |
| 🔍 **빠른 조회** | 거래처별, 기간별 거래 내역 실시간 확인 | 즉시 조회 |
| 📦 **품목 관리** | 자주 거래하는 품목 등록으로 빠른 입력 | 3초 이내 품목 선택 |
| 📊 **자동 집계** | 외상 잔액, 품목별 통계 자동 계산 | 실시간 계산 |
| 💬 **카톡 공유** | 예쁜 거래 명세서를 카카오톡으로 즉시 전송 | 3초 이내 공유 |
| 💾 **안전한 백업** | 클라우드 기반 데이터 보관 | 자동 백업 |

## 2. 문제 정의 및 솔루션

### 2.1 현재 문제점

#### 문제 1: 수기 장부 관리의 불편함
**증상:**
- 종이 장부 분실 위험
- 계산 오류 발생
- 과거 기록 찾기 어려움
- 품목별 집계 수작업 필요

**영향:**
- 거래처와의 분쟁 발생
- 정산 시간 과다 소요
- 매출 추이 파악 불가

#### 문제 2: 엑셀 관리의 한계
**증상:**
- 모바일에서 입력 불편
- 거래처별 집계 수동 작업 필요
- 실시간 동기화 불가
- 품목별 분류 어려움

**영향:**
- 입력 누락 발생
- 데이터 정합성 문제
- 통계 산출 어려움

#### 문제 3: 기존 ERP의 문제
**증상:**
- 소상공인에게 과도한 비용
- 복잡한 기능으로 학습 곡선 높음
- 모바일 최적화 부족
- 단순 거래 기록에는 과도한 스펙

**영향:**
- 사용 포기
- 투자 대비 효과 낮음

### 2.2 솔루션

**간단하고 직관적인 모바일 우선 거래장 앱**

| 기존 방식 | 거래의장인 | 개선 효과 |
|-----------|-----------|----------|
| 수기 장부 작성 | 모바일 터치 입력 | 입력 시간 70% 단축 |
| 수동 계산 | 자동 잔액 계산 | 계산 오류 0% |
| 장부 사진 촬영 → 카톡 전송 | 자동 영수증 생성 → 카톡 공유 | 공유 시간 80% 단축 |
| 품목별 수기 분류 | 품목 선택 입력 | 분류 시간 90% 단축 |
| 종이 보관 | 클라우드 자동 백업 | 분실 위험 0% |

## 3. 핵심 기능 요구사항

### 3.1 MVP (Phase 1) 필수 기능

#### 3.1.1 거래처 관리

**기능 설명**: 거래처 정보 등록 및 관리

**요구사항**:
- ✅ 거래처 등록 (상호명, 담당자, 연락처, 주소)
- ✅ 거래처 목록 조회 (검색, 정렬)
- ✅ 거래처 상세 정보 수정
- ✅ 거래처 삭제 (비활성화)
- ✅ 거래처별 잔액 표시

**화면 구성**:
- 거래처 목록 화면
- 거래처 등록/수정 화면
- 거래처 상세 화면

**우선순위**: 🔴 High (MVP 필수)

---

#### 3.1.2 품목 관리 ⭐

**기능 설명**: 자주 거래하는 품목 등록 및 관리

**요구사항**:
- ✅ 품목 등록 (품목명, 코드, 카테고리, 기본 단가, 단위)
- ✅ 품목 목록 조회 (카테고리별 그룹화)
- ✅ 품목 수정
- ✅ 품목 삭제 (비활성화)
- ✅ 품목별 기본 단가 설정

**사용 시나리오**:
```
1. 사용자가 자주 거래하는 품목 등록
   예: "한라봉" - 과일 카테고리 - 8,000원/kg

2. 거래 입력 시 품목 선택
   → 자동으로 단가 입력됨
   → 수량만 입력하면 금액 자동 계산

3. 거래 내역에 품목 정보 표시
   → 품목별 매출 추이 파악 가능
```

**화면 구성**:
- 품목 목록 화면
- 품목 등록/수정 화면

**우선순위**: 🔴 High (MVP 필수)

**Phase 1 포함 이유**:
- 실제 사용자들이 품목별 관리를 가장 많이 요청
- 품목 없이도 사용 가능하지만, 있으면 입력 효율 3배 증가
- 향후 재고 관리, 품목별 통계 기능의 기반

---

#### 3.1.3 거래 기록 관리

**기능 설명**: 일일 거래 내역 입력 및 조회

**요구사항**:

**생성 (Create):**
- ✅ 거래 등록 (거래처 선택, 거래 유형, 금액, 날짜, 메모)
- ✅ 거래 유형: 받을 돈(매출/입금), 줄 돈(매입/출금)
- ✅ 품목 선택 입력 (선택사항)
  - 품목 선택 → 수량 입력 → 단가 입력 → 자동 계산
  - 또는 금액 직접 입력
- ✅ 빠른 입력 기능 (품목 자동 완성)

**조회 (Read):**
- ✅ 거래 목록 조회 (전체/거래처별/기간별)
- ✅ 거래 내역 검색 기능
- ✅ 거래 상세 정보 확인
- ✅ 품목 정보 표시 (있는 경우)

**수정 (Update):**
- ✅ 거래 내역 수정
- ✅ 품목 정보 수정
- ✅ 금액/날짜/메모 수정

**삭제 (Delete):**
- ✅ 거래 내역 삭제
- ✅ 삭제 확인 다이얼로그 (실수 방지)
- ✅ 삭제 시 잔액 자동 재계산

**화면 구성**:
- 거래 입력 화면 (하단 FAB 버튼)
- 거래 목록 화면
- 거래 상세 화면
- 거래 수정 화면

**우선순위**: 🔴 High (MVP 필수)

**입력 방식 비교**:

| 방식 | 예시 | 장점 | 사용 시나리오 |
|------|------|------|--------------|
| 품목 선택 | 한라봉 10kg × 8,000원 | 자동 계산, 품목별 추적 | 정기 거래 품목 |
| 직접 입력 | 기타 50,000원 | 빠른 입력 | 일회성 거래 |

## 5. 사용자 플로우

### 5.1 최초 사용자 플로우

```
앱 실행
```
